Add Docker build automation and Trivy security scanning for base images

## Overview
- Created Windows batch scripts for automated Docker image builds
- Integrated Trivy vulnerability scanning into build workflow
- Optimized TensorFlow base image (reduced from 18GB to 8-10GB)
- Comprehensive documentation with security best practices
- Added Docker infrastructure overview and usage guides

## New Features

### Build Automation (Windows)
- build-images.bat: Build all base images with auto-scanning
- build-single.bat: Build individual images with validation
- scan-images.bat: Standalone security scanning tool
- verify-images.bat: Check which images are present
- clean-images.bat: Remove images with confirmation prompts

### Security Scanning with Trivy
- Automatic scanning after successful builds
- Focus on CRITICAL and HIGH severity vulnerabilities
- Reports saved to scan-reports/ directory (gitignored)
- Graceful handling when Trivy not installed
- Exit code 1 for CI/CD integration
- Console output with summary and detailed reports

### TensorFlow Image Optimization
- Removed Hugging Face transformers/datasets (~3GB saved)
- Removed cellpose biomedical imaging (~500MB-1GB saved)
- Combined RUN statements and aggressive cleanup
- Added GPU/CPU version switching option
- Result: 18GB → 8-10GB (GPU) or 4-5GB (CPU)

## Usage

### Build All Images
```cmd
cd infrastructure/docker/base-images
build-images.bat
```

### Build Single Image
```cmd
build-single.bat python-base
```

### Scan Existing Images
```cmd
scan-images.bat
```

### Verify Images
```cmd
verify-images.bat
```
